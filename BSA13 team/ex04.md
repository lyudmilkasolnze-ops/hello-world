### Упражнение 04: Оценка результата

Для UC02 (настройка расписания и отправка напоминаний) рассмотрены результаты выполнения позитивного тест-кейса и составлены отчеты о дефектах.

---

#### Оценка результатов после выполнения п.п. 1, 2

##### Ситуация 1.1
> Система сообщает, что выбранный слот забронирован в системе клиентом. Однако при обращении к клиенту тот отрицает, что забронировал этот слот.

* **Результат тест-кейса:** ❌ Отрицательный
* **ФР:** Отрицательный результат. Дефект зарегистрирован — **DEF-0001**

##### Ситуация 1.2
> Система сообщает, что выбранный слот не забронирован в системе клиентом.

* **Результат тест-кейса:** ✅ Положительный
* **ФР:** Положительный результат. Слот свободен, менеджер может продолжить работу по назначению.

---

#### Оценка результатов после выполнения п.п. 1–5

##### Ситуация 2.1
> Система выполняет закрепление слота за мастером, при дальнейшей работе слот не отражается в списке свободных.

* **Результат тест-кейса:** ✅ Положительный
* **ФР:** Положительный результат. Слот корректно закреплен за мастером и исключен из списка свободных.

##### Ситуация 2.2
> Система не выполняет закрепление слота за мастером. При дальнейшей работе этот слот также можно выбрать из списка свободных.

* **Результат тест-кейса:** ❌ Отрицательный
* **ФР:** Отрицательный результат. Дефект зарегистрирован — **DEF-0002**

---

#### Отчеты о дефектах

##### DEF-0001

| Поле | Значение |
|------|----------|
| **Идентификатор** | DEF-0001 |
| **Краткое описание** | Слот отображается как забронированный клиентом, хотя клиент не выполнял бронирование |
| **Предусловие** | Менеджер авторизован в системе. Клиент ранее был зарегистрирован. Слот был свободен до начала теста. |
| **Шаги воспроизведения** | 1. Открыть расписание мастера. 2. Выбрать свободный слот. 3. Система отображает статус «Забронировано клиентом [ФИО]». 4. Связаться с клиентом — клиент не подтверждает бронирование. |
| **Ожидаемый результат (ОР)** | Слот должен отображаться как свободный, если бронирование не было выполнено клиентом. Статус бронирования должен соответствовать фактическим действиям пользователей. |
| **Фактический результат (ФР)** | Слот отображается как «Забронировано» с привязкой к клиенту, который не совершал данного действия. **Отличие от ОР:** некорректная привязка бронирования к клиенту. |
| **Постусловия** | Вручную освободить слот через панель администратора. Проверить логи на предмет несанкционированных действий. |
| **Окружение** | ОС: Windows 11, Браузер: Chrome 131, Разрешение: 1920×1080 |
| **Приоритет** | 1 (Критический — влияет на корректность бронирования) |
| **Важность** | Критическая — может привести к потере клиентов и некорректному распределению времени мастеров |
| **Описание** | Обнаружена проблема целостности данных: система присваивает бронирование клиенту без его фактического действия. Возможные причины: race condition при одновременном доступе, ошибка в логике привязки сессии к бронированию, некорректная обработка незавершенных транзакций. Требуется анализ логов транзакций и проверка механизма блокировки слотов. |
| **Дополнительные материалы** | Скриншот экрана расписания с некорректным бронированием, выгрузка из журнала операций за тестовый период. |

---

##### DEF-0002

| Поле | Значение |
|------|----------|
| **Идентификатор** | DEF-0002 |
| **Краткое описание** | Слот не закрепляется за мастером после назначения и остается в списке свободных |
| **Предусловие** | Менеджер авторизован. Мастер зарегистрирован в системе. Слот находится в статусе «Свободен». |
| **Шаги воспроизведения** | 1. Открыть расписание мастера. 2. Выбрать свободный слот. 3. Назначить слот мастеру (выполнить закрепление). 4. Вернуться к списку свободных слотов. 5. Убедиться, что слот по-прежнему отображается как свободный. |
| **Ожидаемый результат (ОР)** | После закрепления слот должен исчезнуть из списка свободных и отображаться в расписании мастера как занятый. |
| **Фактический результат (ФР)** | Слот остается в списке свободных после закрепления за мастером. **Отличие от ОР:** операция закрепления не сохраняется, слот доступен для повторного выбора. |
| **Постусловия** | Проверить состояние слота в БД напрямую. При необходимости вручную обновить статус. |
| **Окружение** | ОС: Windows 11, Браузер: Chrome 131, Разрешение: 1920×1080 |
| **Приоритет** | 1 (Критический — блокирует основной бизнес-процесс записи) |
| **Важность** | Критическая — приводит к возможности двойного бронирования одного слота |
| **Описание** | Операция закрепления слота за мастером не фиксируется в базе данных. Возможные причины: ошибка в транзакции записи (commit не выполняется), некорректная работа кэша (отображаются устаревшие данные), отсутствие валидации на серверной стороне. Рекомендуется проверить серверные логи и состояние транзакций в СУБД. |
| **Дополнительные материалы** | Скриншот списка свободных слотов до и после закрепления, лог сетевых запросов (DevTools → Network). |
