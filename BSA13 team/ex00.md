### Упражнение 00: Выделение типов требований

В рамках проекта «Запись на стрижку» выделены следующие группы требований, идентификаторы, атрибуты и статусы.

---

#### 1. Типы требований и идентификаторы

| № | Тип требования | Идентификатор | Описание |
|---|----------------|:---:|----------|
| 1 | Бизнес-требования | **BRQ** | Цели и задачи продукта, бизнес-правила |
| 2 | Требования заинтересованных сторон | **SRQ** | User Stories, Use Cases, описания бизнес-процессов |
| 3 | Функциональные требования к системе | **FRQ** | Спецификации функций системы, интерфейсы |
| 4 | Нефункциональные требования к системе | **NRQ** | Атрибуты качества (производительность, безопасность и др.) |

Формат идентификатора: **XXX-NNNN**, где XXX — трёхбуквенный код типа, NNNN — порядковый номер (4 цифры с ведущими нулями).

Примеры: `BRQ-0001`, `SRQ-0003`, `FRQ-0012`, `NRQ-0005`.

#### 2. Атрибуты требований

##### 2.1. Состояние

| Атрибут | Описание | Пример значения |
|---------|----------|-----------------|
| Дата и время создания | Момент регистрации требования | 28.02.2026 10:00 |
| Дата и время перевода в текущий статус | Момент последнего изменения статуса | 01.03.2026 14:30 |
| Статус требования | Текущее состояние (см. раздел 3) | На анализе |
| Приоритет требования | Срочность реализации (1 — высший, 5 — низший) | 1 |
| Версия продукта | Версия, в которую включено требование | v1.0 |
| Патч (спринт/выпуск) | Итерация, в рамках которой реализуется | Sprint 3 |

##### 2.2. Описание и обоснование

| Атрибут | Описание | Пример значения |
|---------|----------|-----------------|
| Краткое наименование (≤50 символов) | Лаконичное название требования | Автоматическая отправка напоминаний |
| Описание (≤150 символов) | Развернутая формулировка требования | Система отправляет напоминание клиенту через выбранный канал связи за N часов до записи |
| Размещение материалов | Ссылка на документ, схему, макет | [ссылка на Confluence/WEEEK] |

| Источник / Вышестоящее требование | Связь с требованием верхнего уровня | BRQ-0001 |
| Связи требования | Горизонтальные связи с другими требованиями | FRQ-0003, NRQ-0002 |

#### 3. Статусы требований

| Статус | Описание |
|--------|----------|
| **Новое** | Требование зарегистрировано, ожидает рассмотрения |
| **На анализе** | Аналитик прорабатывает и детализирует требование |
| **В разработке** | Требование передано разработчикам для реализации |
| **На тестировании** | Реализация проверяется тестировщиками |
| **Включено в патч** | Требование реализовано и включено в конкретный релиз/спринт |
| **Отклонено** | Требование отклонено (неактуально, дублирует другое) |
| **Отложено** | Реализация перенесена на более поздний срок |
| **Завершено** | Требование полностью реализовано, протестировано и принято |

Диаграмма переходов статусов:

```
Новое → На анализе → В разработке → На тестировании → Включено в патч → Завершено
              ↓               ↓               ↓
          Отклонено       Отложено        Отклонено
```
